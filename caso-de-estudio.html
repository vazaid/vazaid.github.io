Infraestructura Viva

Migración de infraestructura on-premise a arquitectura cloud en AWS

AWS EC2 S3 DynamoDB VPC CloudWatch SNS SQS

Descripción de la actividad

El proyecto Infraestructura Viva consistió en diseñar e implementar una solución completa de migración cloud para la empresa ficticia Soluciones Digitales ACME.

La solución debía cubrir todas las capas de una arquitectura cloud moderna: almacenamiento, bases de datos relacionales y NoSQL, cómputo, red virtual, mensajería y monitoreo.

Desafío principal

El mayor desafío no fue técnico en sí mismo, sino conceptual: comprender por qué cada decisión arquitectónica importa.

  • Restricciones de permisos en AWS Academy: el rol voclabs no permitía crear instancias RDS ni CloudFront.
  • Diagnóstico de errores: el servidor EC2 no cargaba porque el navegador forzaba HTTPS al puerto 443.
  • Gestión de servicios interdependientes: coordinación entre S3, DynamoDB, EC2, SNS, SQS y CloudWatch.

Solución propuesta

Almacenamiento

S3 para archivos activos y Glacier para archivado automático.

Bases de datos

RDS MySQL para datos transaccionales y DynamoDB para tickets de soporte.

Cómputo

Instancia EC2 con Apache preparada para escalado horizontal.

Red

VPC con subredes públicas y privadas en dos zonas de disponibilidad.

Mensajería y monitoreo

SNS, SQS y CloudWatch para alertas, colas de mensajes y monitoreo.

Principales aprendizajes

Arquitectura cloud: seguridad, escalabilidad y disponibilidad dependen de decisiones de diseño.

Resolución de problemas: diagnosticar capa por capa fue clave para identificar errores.

Documentación técnica: el proyecto se documentó completamente en Markdown con evidencia técnica.
